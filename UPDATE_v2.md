# 系统更新说明 v2.0

## 更新日期
2025-01-08

## 更新内容

### 1. 所有政策引用添加官方链接 ✅

**更新前**：
- AI回复中的政策引用没有链接
- 用户无法自行核实

**更新后**：
- 所有政策引用都包含可点击的官方链接
- 每个数据都有来源标注
- 链接格式：[政策名称](http://...)

**示例**：
```
【推荐依据】：
1. 政策条款：《北京市基本养老保险规定》第十七条
   官方链接：http://rsj.beijing.gov.cn/xxcx/zxyc/yldx/
   具体内容：基础养老金计算公式
```

### 2. 改为逐个问题提问 ✅

**更新前**：
- 一次问多个问题（3-5个）
- 用户容易遗漏

**更新后**：
- 每次只问1个问题
- 等用户回答后再问下一个
- 清晰标注问题进度（第X/14个问题）

**示例**：
```
【第1/14个问题】
请问您的性别是？

相关政策：
- 法定退休年龄规定：http://rsj.beijing.gov.cn/xxcx/zxyc/yldx/
```

### 3. 增加信息确认环节 ✅

**更新前**：
- 问完问题直接计算
- 没有确认机会

**更新后**：
- 问完14个问题后，给出完整信息总结
- 等用户确认无误后才开始计算
- 用户可以修改任何信息

**流程**：
```
1. 逐个询问14个问题
2. 给出【信息总结】
3. 请求用户确认
4. 用户确认后才开始详细分析
```

### 4. 所有判断必须有具体条款支持 ✅

**更新前**：
- 部分判断缺少政策依据
- 推荐理由不够具体

**更新后**：
- 每个判断都要引用具体条款号
- 每个计算都要注明政策来源
- 每个推荐都要有详细依据

**格式要求**：
```
【推荐档位】：XXX档

【推荐依据】：
1. 政策条款：《XX规定》第X条
   官方链接：http://...
   具体内容：...

2. 计算依据：公式XXX
   政策来源：http://...
   您的具体情况：...

3. 回本分析：
   - 数据来源：官方公告XXX
   - 计算过程：...
   - 结论：...
```

### 2. 政策背景信息增强

`prompts/policy_context.txt` 文件大幅扩展：
- 重要官方网站列表
- 关键时间节点（1992年、1998年、2025年）
- 养老金计算三要素详细说明
- 法定退休年龄规定
- 视同缴费年限认定
- 4050政策说明
- 特殊情况处理

### 3. 问答流程规范化

新的对话流程：

```
阶段1：自我介绍
- 介绍系统功能
- 说明14个问题流程
- 提供官方政策网站

阶段2：逐个提问（14个问题）
- 第1-5个问题：基础信息
- 第6-10个问题：工作经历
- 第11-12个问题：收入情况
- 第13-14个问题：其他信息

阶段3：信息总结
- 列出所有收集的信息
- 请求用户确认
- 允许修改

阶段4：详细分析
- 计算过程（每步有政策依据）
- 三档对比表格
- 回本分析
- 最终推荐建议（具体条款支持）
```

## 注意事项

### 链接格式
- 使用Markdown格式：[文字](URL)
- 确保链接可点击
- 链接必须是官方网站

### 政策时效性
- 提示用户查看最新政策
- 提供官方查询渠道
- 建议咨询社保局

### 不确定性处理
- 不确定的信息不要猜测
- 宁可问用户也不要假设
- 明确标注假设和估算
